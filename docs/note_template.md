# Шаблон заметки (1–2 страницы)

Вам нужно кратко зафиксировать свои решения, чтобы их можно было объяснить и проверить. Текст не должен быть длинным — цель не в “документации ради документации”, а в ясности.

## 1) Что реализовано
- Пакеты: internal/skiplist (ядро), internal/kv/memskiplist (адаптер под kv.Store).
- Операции: Put, Get, Delete, Scan с ленивым итератором.

## 2) Инварианты и семантика
- Ключи упорядочены лексикографически (bytes.Compare).
- Put существующего ключа — обновление, не дубликат.
- Scan([start, end)) — полуоткрытый интервал; nil = ±∞; start >= end → пусто.
- Итератор ленивый.
- Get отсутствующего ключа → ErrNotFound.
- Delete отсутствующего: skiplist → ErrNotFound, memskiplist → nil (идемпотентен).
- Key/value копируются на Put.

## 3) Принятые решения (и почему)
- maxLevel=32 — запас до 4 млрд ключей, константа для простоты.
- coin-flip генерация уровня — нагляднее табличного варианта.
- p=0.5 — компромисс скорость/память.
- Копирование на Put — безопасность при переиспользовании буферов, идиоматично для Go.
- Локальный rng с seed — воспроизводимость тестов.
- Итератор как структура — нужен Close() и состояние.
